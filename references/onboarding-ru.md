# Онбординг — Telegram Codex Cockpit

Это инструкция для ежедневной работы в Telegram Codex cockpit.

## Коротко
- У тебя есть отдельный Telegram-чат под Codex
- Внутри есть topics
- **Один topic = одна задача / один поток работы**
- Первый bind делается через `/cas_resume`
- После bind можно писать обычным текстом

## Что где делать
### `Codex Control`
Используй для:
- первого bind
- проверки статуса
- управления текущим Codex thread
- теста кнопок и `/cas_*`

### `Inbox / Triage`
Используй для:
- новых задач, которые ещё не разложены по темам
- быстрых входящих идей

### Остальные topics
Используй как отдельные рабочие зоны под проекты, инфраструктуру, продукт или исследования.

## Самый правильный первый запуск
1. Зайди в `Codex Control`
2. Отправь `/cas_resume`
3. Если предложит `Existing` или `New`, для первого запуска обычно выбирай `New`
4. После bind напиши обычную задачу, например:
   - `Проверь репозиторий, составь план и начни с безопасных правок`

## Какие команды реально нужны
### Основные
- `/cas_resume` — привязать topic к Codex thread
- `/cas_status` — посмотреть статус
- `/cas_plan` — сначала получить план
- `/cas_stop` — остановить текущий run
- `/cas_review` — попросить review
- `/cas_model` — сменить модель

### Когда нужна `/cas_steer`
Используй, если агент пошёл не туда:
- `/cas_steer сначала локализуй проблему, архитектуру не трогай`
- `/cas_steer работай только в этом каталоге`
- `/cas_steer сначала дай короткий план, потом делай`

## Как работать без боли
1. Один раз сделал bind через `/cas_resume`
2. Дальше просто пишешь по-человечески
3. Для новой большой задачи открываешь отдельный topic

## Как устроены кнопки
Кнопки появляются **в ответе бота**, а не сами по себе.

Чтобы получить кнопки:
1. отправь `/cas_resume` или `/cas_status`
2. дождись ответа бота
3. нажимай кнопки уже в этом сообщении
